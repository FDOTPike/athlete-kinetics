**Root Cause / Current State:**
Crash-on-open caused by native-module initialization executed at JS module-evaluation time (bundle init), i.e. before any try/catch or React error boundary can exist. Three import-time initializers sat in the boot chain:
1. `@op-engineering/op-sqlite` (`src/functions.ts`): top-level `NativeModules.OPSQLite.install()` with explicit `throw new Error(...)` on null module / failed JSI install / missing `global.__OPSQLiteProxy`. Reached via `App.tsx -> useStore.ts -> @ak/core-db pragmas.ts` static import.
2. `onnxruntime-react-native@1.24.3` (`lib/binding.ts`): top-level side effect `Module.install()`; TypeError if `NativeModules.Onnxruntime` is null. Additionally its Android `OnnxruntimeModule.install()` calls `getReactApplicationContext().getCatalystInstance().getJSCallInvokerHolder()` — a legacy-architecture API unavailable under RN 0.81 bridgeless (`newArchEnabled=true` in `apps/mobile/android/gradle.properties`), so its JSI install cannot succeed on this build regardless; failure mode is contained (returns false -> throwing Proxy) only if the NativeModules lookup itself succeeds.
3. `react-native-blob-util@0.24.9`: native-module access on import (same class of hazard).
Exact culprit on-device not yet confirmed by logcat; all three are eliminated as crash vectors by the fix below. Exclusions: closed PR `copilot/optimize-fable-5-performance` was never merged (`master` == APK commit 5d71f0f); JS-bundle content verified complete (schema SQL, policy, codebase, vectors, tokenizer present in 1.7 MB Hermes-input bundle).
